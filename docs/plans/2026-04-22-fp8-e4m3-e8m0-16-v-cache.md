# FP8 E4M3 E8M0 16 V Cache Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add non-flash CUDA V cache support for `fp8_e4m3_e8m0_16`, with V and P both quantized in block-16 FP8 E4M3 plus E8M0 scale format.

**Architecture:** Add a ggml quantized type for block16 FP8. Reuse the existing FP8 cache layout decisions, but route non-flash `V * P` through a native CUDA cuBLASLt path after quantizing P to the same block16 format.

**Tech Stack:** C/C++, CUDA, ggml quantization traits, llama.cpp KV cache graph construction, CTest CUDA test target.
